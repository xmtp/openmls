Struct openmls::framing::MlsMessageIn

pub struct MlsMessageIn { /* private fields */ }
Expand description

Before use with the MlsGroup API, the message has to be unpacked via extract to yield its MlsMessageBodyIn.

// draft-ietf-mls-protocol-17
struct {
    ProtocolVersion version = mls10;

    // ... continued in [MlsMessageBody] ...
} MLSMessage;

The -In suffix of this struct is to separate it from the MlsMessageOut which is commonly returned by functions of the MlsGroup API.
